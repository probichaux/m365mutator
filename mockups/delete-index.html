The Deletions page (mod05)

Three directions for the new Deletions tab. All three share the same three-part layout — Users (the standard Explicit/Random target panel, reused unchanged), Workloads (Mail / Calendar / OneDrive), and a date scope (everything · after · before · between). They differ only in how the workload picker and date scope are presented. Every variant ends with a live plain-English summary and an explicit acknowledgement gating the destructive Delete action.

  • 1 — Inline scopeWorkload checkboxes in a row; scope is a pill row with date fields that appear inline only when needed. The most compact.
  • 2 — Choice cardsWorkloads as toggle cards; scope as descriptive radio cards that explain each option in words before you enter a date. The most explanatory.
  • 3 — Sentence builderWorkloads as toggle chips; scope reads as one editable sentence ("Delete everything after [date]…") so the rule is unambiguous. The most novel.
Chosen direction: 2 — Choice cards. Scope semantics are settled: items match on the date they arrived (received date for mail, created date for calendar events and files), and the boundary dates are included (on-or-after / on-or-before).
